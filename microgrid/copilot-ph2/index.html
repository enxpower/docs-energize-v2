Phase 2 Copilot

Interactive EMS Modules Phase 2

Five interconnected modules demonstrating the EnergizeOS AI + deterministic EMS architecture. Each module runs a live simulation seeded with real C&I BESS parameters. The AI Copilot panel reads the EMS state and answers operational questions — it never issues device commands.

F1

Zero Export / Reverse Flow Guard

Interactive simulation of the PCC power flow control loop that prevents net export to the utility under CA Rule 21. Adjust site load and BESS dispatch with live reverse-flow detection, dead-band enforcement, and automatic clamping.

Live simulation SDG&E Rule 21
F2

Grid ↔ Island ↔ Grid State Machine

Walk through the complete 7-state grid-island-grid sequence. Trigger grid loss, watch BESS transition to grid-forming mode, simulate grid restoration, and step through the synchronized reclose sequence including trip lockout behavior.

State machine IEEE 1547-2018
F3

C1–C7 Close Interlock Diagnostics

Real-time C1–C7 permissive chain status with per-condition source, data age, and blocking reason. Inject communication faults, BESS faults, sync failures, and spring-charge failures to see how the EMS responds.

Interlock chain C1–C7 logic
F4

TOU / Demand Charge Optimizer

Day-ahead BESS dispatch plan optimized for SDG&E TOU-DR-OPT. See 24-hour energy and demand savings breakdown, peak shaving strategy, and how SOC constraints interact with on-peak windows.

Optimization TOU-DR-OPT
Architecture note: All five modules share a single deterministic EMS engine and site data model. The AI Copilot reads a state snapshot on every query — it never modifies engine state or issues device commands. This mirrors the production architecture described at docs.energizeos.com/microgrid/architecture.
