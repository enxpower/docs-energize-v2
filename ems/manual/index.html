Section 01

Executive Summary

EnergizeOS™ is a commercial-grade, hardware-software integrated Energy Management System for industrial and commercial (C&I) energy storage, microgrid, and grid-interconnection applications.

Developed by Energize Solutions Inc., EnergizeOS™ integrates a ruggedized industrial control panel (ECP), a real-time embedded strategy execution engine, and an OTA management platform into a standardized, production-ready control framework.

ℹ
EnergizeOS™ is not an HMI and not a project-specific EMS script. It is a productized control platform with defined engineering responsibility boundaries, built-in interlock logic, compliance-ready architecture, and full lifecycle support — designed to deploy repeatedly across diverse sites and jurisdictions without re-engineering.

2.2 Control Philosophy

EnergizeOS™ is governed by a formally defined control philosophy that establishes unambiguous engineering boundaries between protection and control responsibilities. These principles are immutable — they are not configurable.

// EnergizeOS™ Immutable Control Principles Protection Trip = Relay Primary // protection relay owns protection engineering responsibility Operational Trip = EMS Optional // controlled disconnection for commissioning / maintenance only Breaker Close = EMS Exclusive // no blind close; no auto-reclose by default Close Gate = C1–C7 All Met // all seven interlock conditions must be satisfied simultaneously Feedback = Physical 52a/52b // completion confirmed by hardware, not by command issuance Measurement Gap = Block High-Risk // stale or conflicting data blocks close; never acts on incomplete data
⚠
Responsibility separation is fundamental. Protection responsibility — including protection relay settings, protection coordination, and fault current analysis — remains with the protection relay, site protection design, and licensed protection engineers. EnergizeOS™ does not overlap with or absorb protection engineering responsibility. The EMS holds control and supervisory responsibility only.

4.2 C1–C7 Breaker Close Interlock Chain

All PCC breaker close operations must satisfy seven conditions simultaneously. This chain is enforced in hardware (52b DI, sync-check relay output) and software (EMS logic, role check) in series. No individual condition can be bypassed without administrator intervention and a logged override event.

CodeConditionTechnical SpecificationData Source
C1 Grid Stable ≥ 60 s Voltage ∈ [210–260 V L-N]; Frequency ∈ [59.5–60.5 Hz]; both parameters within range simultaneously for ≥ 60 continuous seconds Anti-islanding relay measurement / utility-grade meter
C2 Zero Export Verified PCC active power < ±50 W; power direction = 0 (no net injection to utility); prevents close into export-prohibited state Revenue-grade meter (Modbus TCP)
C3 BESS Idle or Charging PCS operational state = Idle or Charging only; no active discharge setpoint; prevents close with DER output active PCS controller status register
C4 Close Authorized Administrator has confirmed via UI close button; active session role = Admin; prevents unauthorized or accidental close Local or Remote UI (role module)
C5 Phase Sync Completed Phase angle difference between island bus and utility ≤ 5°; sync-check relay output confirmed; prevents out-of-phase close Phase synchronizer / sync-check relay DI output
C6 Breaker Currently Open 52b auxiliary contact = OPEN confirmed via hardwired DI; prevents duplicate close command on already-closed breaker Breaker 52b auxiliary contact (hardwired DI)
C7 Admin Role Verified Active logged-in user = Administrator role; operator-level accounts cannot initiate close; enforces operational discipline UI role and session module
⚠
If any single condition C1–C7 is not satisfied, the Close button is locked and disabled in the UI. An on-screen advisory identifies the specific blocking condition. The blocking event is logged with timestamp, condition code, and measured values. No close command is issued to the breaker DO output.

Section 05

Protection, Redundancy & Fail-Safe Design

5.1 Dual-Path Anti-Islanding Protection

EnergizeOS™ implements a dual-path anti-islanding architecture compliant with UL 1741 SB, IEEE 1547-2018, CPUC Rule 21, and EU EN 50549. Both paths operate simultaneously and independently. Grid reconnection is blocked until both paths confirm stable grid conditions.

PathHardwareDetection ParametersTrip ExecutionIndependence
Path A (Primary) Dedicated anti-islanding relay (e.g., ABB CM-UFD, SEL-700G, SEL-735) V magnitude deviation; frequency out-of-range; ROCOF threshold; anti-islanding algorithm Direct hardwired trip to breaker trip coil — no EMS software in the trip path Hardware-only; operates independently of EMS software state
Path B (Secondary) Revenue-grade smart meter (Modbus TCP, e.g., eGauge) Voltage presence; current direction; active power import/export direction EMS issues operational trip via DO when meter data indicates grid-loss condition Software path; secondary to relay; provides EMS visibility and log attribution
ℹ
Path A is the primary anti-islanding protection and must satisfy utility interconnection requirements (IEEE 1547-2018 §8, UL 1741 SB). Path B provides EMS-layer visibility and secondary action. Path A relay settings and protection coordination are the responsibility of the site protection engineer — not EnergizeOS™.

5.2 Triple-Redundant Breaker Trip Architecture
