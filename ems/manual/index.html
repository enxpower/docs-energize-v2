Section 01

Executive Summary

EnergizeOS™ is a commercial-grade, hardware-software integrated Energy Management System for industrial and commercial (C&I) BESS, microgrid, and grid-interconnection applications.

Developed by Energize Solutions Inc., EnergizeOS™ integrates a ruggedized industrial control panel (ECP), a real-time embedded strategy execution engine, and an OTA management platform into a standardized, production-ready control framework. It is not a project-specific EMS script. It is a productized, auditable, repeatable control platform with defined engineering responsibility boundaries at every layer.

⚡
Core design principle: EnergizeOS™ separates protection responsibility (relay) from control responsibility (EMS). This boundary is immutable and enforced in hardware and software at every deployment.

2.2 Control Philosophy (Immutable)

Protection Trip = Relay Primary // relay owns protection; EMS observes and logs only Operational Trip = EMS Optional // controlled disconnection for commissioning/maintenance only Breaker Close = EMS Exclusive // no blind close; no auto-reclose by default Close Gate = C1–C7 All Met // all 7 conditions must be satisfied simultaneously Feedback = Physical 52a/52b // completion confirmed by hardware DI, not by command issuance Comm Timeout = Safe Frozen State // Modbus loss ≥5s → freeze setpoints, invalidate C1, raise P1 alarm PCS Island Mode = Grid-Forming // EMS commands mode-switch before island; verifies confirmation SOC Threshold = ±2% Hysteresis // deadband prevents charge/discharge oscillation Priority Order = Safety>Protection>Grid>Storage>Economy
⚠
Responsibility separation is fundamental. Protection relay settings, coordination studies, and fault-current analysis remain with the site protection engineer. EnergizeOS™ holds control and supervisory responsibility only.

2.3 Grid-Forming vs. Grid-Following

The most commonly misconfigured parameter in BESS/microgrid commissioning. EnergizeOS™ explicitly manages PCS operating mode as part of the grid-tie/island transition sequence.

PCS ModeWhen ActiveV/f SourceEnergizeOS™ Action
Grid-FollowingGrid-connectedUtility grid sets V and f; PCS injects currentNormal dispatch: charge/discharge setpoint via Modbus
Grid-FormingIslandedPCS establishes V and f reference for island busIssues mode-switch command on island; verifies confirmation within 2s; alarm if PCS fails to confirm
Grid-Forming + DroopMulti-PCS islandEach PCS shares load via P-f and Q-V droopConfigures droop parameters; monitors frequency deviation for instability
⚠
Critical SAT verification: Grid-forming mode must be tested under load during SAT. A PCS left in grid-following mode cannot island. This failure is silent during grid-connected operation and discovered only at the first actual grid outage.

4.3 C1–C7 Breaker Close Interlock Chain

All PCC breaker close operations require all seven conditions to be satisfied simultaneously, enforced in hardware and software in series.

Code
Condition
Technical Definition
Data Source
C1
Grid Stable ≥ 60 s
V ∈ [210–260 V]; f ∈ [59.5–60.5 Hz] sustained ≥ 60 s continuously
Voltage AND frequency both in range for 60 continuous seconds — not just at moment of check
Anti-islanding relay / revenue meter
C2
Zero Export Verified
Active power < ±50 W; no reverse injection to utility PCC
Prevents close into export-prohibited state; measured at PCC not at inverter terminals
Revenue-grade meter (Modbus TCP)
C3
BESS Idle or Charging
PCS state = Idle or Charging only; no active discharge setpoint
Prevents close with DER output active into grid; avoids out-of-phase injection

C4

Admin confirmed via UI; role and session verified
Prevents unauthorized or accidental close; role check independent of session timeout
UI role and session module
C5
Phase Sync Completed
Phase angle ≤ 5° between island bus and utility; sync relay confirmed
Prevents out-of-phase close that could damage generators, motors, and bus equipment
Sync-check relay DI output (hardwired)
C6
Breaker Currently Open
52b auxiliary contact = OPEN confirmed via hardwired DI
Prevents duplicate close command on already-closed breaker; indeterminate 52a+52b state blocks close

UI role module
⚠
If any single C1–C7 condition is not satisfied, the Close button is locked in the UI, an advisory identifies the blocking condition, and the event is logged with timestamp and condition state. No close command is issued to the DO output.
