The Problem

Why EMS deployments fail.

Not edge cases. The reasons projects go over budget, fail utility inspection, or get called back on site.

01
Blind breaker closes
Most EMS platforms issue a close command and assume execution. No 52a/52b feedback loop, no sync-check, no phase-angle verification. Failure appears only in relay event reports — after the event.
"We had an auto-reclose that took the feeder down. The EMS had no record of it."
02
No audit trail for utilities
Utilities require complete event logs — every trip, close, and mode change with source attribution and timestamps. Project-specific scripts don't satisfy SDG&E, SCE, or CPUC Rule 21 requirements.
"The interconnection engineer asked for 6 months of logs. We had a partial CSV."
03
Every project re-engineered
Custom EMS logic for Site A doesn't transfer to Site B. Different hardware, different utility — the engineering cycle restarts. No standard. No repeatable delivery. No product.
"We quoted 3 weeks for commissioning. It took 11 weeks and two extra site visits."

C1–C7 Interlock Chain

No blind closes.
Every single time.

All PCC breaker close operations require all seven conditions simultaneously — enforced in hardware and software. Cannot be bypassed.

Grid Stable ≥ 60 s
V 210–260 V; f 59.5–60.5 Hz sustained ≥ 60 s
← Anti-islanding relay / revenue meter

← Revenue-grade meter (Modbus TCP)

← PCS status register

← UI role module

Phase angle ≤ 5°; sync-check relay confirmed
← Sync-check relay DI (hardwired)

Active session = Administrator; operator rejected
← Session / permission module
4 conditions not satisfied. Close command blocked.
Protection stays with the relay.
EnergizeOS™ does not absorb protection engineering responsibility. Relay settings, coordination studies, and fault-current analysis stay with the site protection engineer. The EMS holds close authorization only.
Triple-redundant trip. Hardware primary.
Path 1: EMS DO → trip coil. Path 2: UVT (hardware, fires on UPS collapse — no software involved). Path 3: Manual field button (hardwired, always available). All three paths are independent with no common failure mode.
SOC hysteresis prevents oscillation.
±2% deadband on all SOC thresholds prevents charge/discharge hunting — a common root cause of PCS contactor failure in production deployments. Without hysteresis, a BESS at the SOC floor toggles state at the control loop frequency.
Comm loss → safe frozen state.
Modbus timeout to any critical device ≥ 5 s → setpoints frozen, C1 invalidated, P1 alarm raised. The system never closes on stale or absent measurement data. Never.
