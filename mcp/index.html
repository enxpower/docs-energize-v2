为什么 EnergizeOS 需要 MCP 接口

一个 BESS 项目每天产生数百万个数据点——SOC 曲线、功率流向、保护动作日志、电网事件告警。这些数据今天的使用方式是：工程师盯着 HMI 看，事后手动写运维报告，按月生成合规文档。

MCP（Model Context Protocol）是 Anthropic 在 2024 年末发布的开放标准，让 AI 模型能够通过标准化接口调用外部工具和数据。它不是 ChatGPT 插件，也不是简单的 API wrapper——它是一套双向、持久化、可审计的 AI-to-System 通信协议。

对 EnergizeOS 而言，MCP 的价值在于：把我们在 EMS 里已经掌握的现场数据、调度逻辑和合规知识，变成 AI Agent 可以直接理解和调用的能力。结果是：

运维效率
工程师用自然语言问"昨晚 BESS 为什么切换到孤岛模式"，30 秒得到完整根因分析，原来需要 30 分钟人工排查。
合规自动化
CA Rule 21 互联合规报告、ITC 税收抵免运行记录，AI 自动从时序数据库提取、计算并生成报告草稿，人工只需审阅。
客户自助服务
PepsiCo 设施经理可以直接问"这个月 demand charge 节省了多少"，不需要等 EnergizeOS 工程师出具报告。
第三方开放
EPC 商、运维公司、能源交易商可通过 API Key 接入您管理的项目数据，构建自己的 AI 应用，并向 EnergizeOS 付费。

系统架构

整套方案分四层，每层职责清晰。MCP Server 是第三层，是 EMS 与 AI 世界之间的唯一受控出口。

💡
关键设计原则：MCP Server 是 EMS 数据的唯一受控出口。所有 AI 调用必须经过 API Key 认证、权限检查和操作日志记录，AI 永远无法直接接触 Modbus/DNP3 总线或现场设备。

工具接口清单

以下是 MCP Server 向 AI 暴露的工具集。工具设计原则：读操作无限制，写操作强制人工审批，报告生成异步执行。

read_site_status 只读
返回指定站点的实时运行状态，包括 SOC、当前充放电功率、电网连接状态、活跃告警列表。
site_id time_range include_alarms
get_power_flow 只读
获取指定时间段内的功率流向数据，支持 1 秒到 15 分钟多粒度。可用于根因分析和异常检测。
site_id start_time end_time resolution
query_savings 只读
计算指定周期内的 demand charge 节省额、峰谷套利收益和 demand response 参与收益，返回分项明细。
site_id billing_period breakdown
list_events 只读
查询运行事件日志，包括保护动作、模式切换、通信中断、电网事件。支持按严重等级和时间范围过滤。
site_id severity event_type limit
generate_report 报告
异步生成合规报告，支持 CA Rule 21 互联报告、ITC 税收抵免运行记录、SDG&E 月度汇报。返回草稿 PDF 链接。
report_type period site_ids format
forecast_load AI 推理
基于历史负荷模式 + 天气数据，预测未来 24–72 小时负荷曲线，并推荐最优 BESS 调度策略。
site_id horizon_hours weather_source
dispatch_command 写操作 ⚠
生成调度指令草稿（充电 / 放电 / 模式切换 / setpoint），不直接执行，返回待审批工单，需工程师在 HMI 或管理后台确认后方可下发。
site_id command_type value require_approval: true
get_interlock_status 只读
返回当前 C1–C7 联锁链状态，包括各级联锁条件是否满足、上次触发时间和触发原因，用于安全检查和故障分析。
site_id chain_level
⚠️
写操作安全原则：dispatch_command 在第一阶段只生成指令草稿，不连接到任何控制通道。真实下发须通过独立的人工审批流程，与 MCP 接口完全解耦。这是工业系统 AI 接入的基本安全边界。

安全控制边界

工业控制系统接入 AI 的最大顾虑是安全边界。EnergizeOS MCP 从设计上就是物理隔离的数据代理层，AI 永远无法直接触达现场设备。

🔐
API Key 认证
每个接入方拥有独立 API Key，支持细粒度权限控制。只读 Key 无法调用任何写操作工具。
📋
完整审计日志
所有工具调用记录 timestamp、调用方身份、入参和返回结果，不可篡改，保留 90 天。
🚧
写操作强制审批
调度指令只生成草稿，下发须通过独立审批通道，与 MCP 接口完全解耦，无法绕过。
🌐
TLS 1.3 全程加密
MCP 通信全程 TLS 1.3 加密，证书锁定，防止中间人攻击。满足 NERC-CIP 框架要求。
🏗️
网络层隔离
MCP Server 部署在独立的 DMZ 网段，与 OT 网络之间通过单向数据二极管或受控防火墙隔离。
📦
速率限制
每个 API Key 按分钟和小时设置调用上限，防止 AI Agent 误操作导致数据库压力异常。

真实使用场景

以下对话示例展示了接入 MCP 后，工程师和客户如何用自然语言与 BESS 系统交互。

01
运维工程师：事故根因分析
工程师
昨晚 11:37 CA-PEP01 站点切换到孤岛模式，具体是什么触发的？当时 BESS 的 SOC 和功率状态是什么？
Claude（调用 list_events + get_power_flow）
根因分析：CA-PEP01 / 2026-06-09 23:37:14 PST

触发原因：SDG&E 主网检测到 0.4Hz 频率偏差，超过 CA Rule 21 联锁阈值 (±0.2Hz)，EMS C3 联锁条件满足，自动切换到孤岛运行模式。

切换时刻状态：SOC 72.4%，BESS 以 -340 kW 充电（接收电网电力），切换后立即转为 +280 kW 放电维持厂区负荷。

持续时间：23:37 至 00:12（共 35 分钟），00:12:08 电网频率恢复至 59.98Hz，自动并网成功。

建议：此次孤岛切换流程正常，无需处理。建议在月度报告中记录此事件用于 SDG&E 互联合规报备。
02
PepsiCo 设施经理：账单核查
设施经理
这个月 BESS 帮我节省了多少 demand charge？跟上个月比怎么样？
Claude（调用 query_savings）
CA-PEP01 / 2026 年 5 月账单节省明细

Demand Charge 节省：$18,420（峰值削减 214 kW，SDG&E TOU-PA-3 费率）
峰谷套利收益：$4,280
需求响应参与：$1,150（参与 2 次 SDG&E DR 事件）
合计：$23,850

对比上月（4 月）$19,340，本月增长 +23.3%。主要原因：5 月天气升温，用电高峰时段延长，BESS 放电利用率从 68% 提升至 81%。
03
项目经理：ITC 合规文件准备
项目经理
Q1 的 ITC 税收抵免申请需要 BESS 运行记录，帮我生成一份符合 IRS 要求格式的报告。
Claude（调用 generate_report）
已提交报告生成任务（异步处理，预计 3 分钟完成）。

报告将包含：Q1 累计充放电量（MWh）、月均可用率（99.1%）、充放电循环次数（91 次）、SOC 管理合规记录，格式符合 IRS Form 3468 附件要求。

完成后发送至：andy.gong@energizeos.com
PDF 下载链接将在此处更新。

注意：此为草稿，请在签署前由税务顾问审阅最终数据。

合规报告自动化

合规报告是目前最耗费工程师时间的重复性工作，也是客户愿意为之付费的最直接价值。以下是 MCP 接口覆盖的主要合规场景：

CA Rule 21 互联合规
自动从 EMS 提取 12 个月互联运行数据，包括频率响应记录、孤岛事件报告、电能质量指标，生成 SDG&E/SCE/PG&E 标准格式提交文件。
年报 / 月报自动格式化
ITC 税收抵免
提取 BESS 充放电日志，计算可再生能源充电比例（≥75% 即满足 ITC 要求），生成 IRS 认可的运行记录附件，显著降低税务审计风险。
IRS Form 3468季度更新
ERCOT / CAISO 电力市场
生成 Ancillary Services 参与记录（频率调节、备用容量），协助验证 BESS 参与 CAISO PDCI 和 ERCOT ECRS 的合规证明文件。
市场结算性能验证
Fortune 500 内部审计
为 PepsiCo、Shell 等企业客户生成 ESG 合规所需的能耗数据报告、碳减排量计算、峰值负荷管理记录，满足内部 EHS 审计要求。

快速接入指南

当 MCP Server 就绪后，接入 Claude 只需两步：配置 MCP 连接，然后开始对话。

Step 1 — 在 Claude 中添加 MCP Server

在 Claude Desktop（MacOS / Windows）的 claude_desktop_config.json 中添加以下配置：

JSON
{
  "mcpServers": {
    "energizeos": {
      "type": "http",
      "url": "https://mcp.energizeos.com/v1",
      "headers": {
        "Authorization": "Bearer YOUR_API_KEY",
        "X-Site-ID": "CA-PEP01"
      }
    }
  }
}

Step 2 — 开始对话

重启 Claude Desktop 后，即可直接用中文或英文与 EMS 数据交互：

自然语言示例
# 运维查询
"CA-PEP01 现在 SOC 是多少？有没有活跃告警？"

# 财务分析
"上个月 demand charge 节省了多少？帮我生成一张账单摘要。"

# 合规报告
"生成 Q2 的 ITC 运行记录报告，格式符合 IRS Form 3468 要求。"

# 故障分析
"查一下昨天所有 Severity ≥ Warning 的事件，帮我分析根因。"
